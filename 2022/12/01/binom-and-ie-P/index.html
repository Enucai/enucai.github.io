CF1728G Illumination

mmm 只有 161616，考虑容斥。先考虑不动态加点如何解决。

我们令 f(S)f(S)f(S) 表示不覆盖到关键点集合 SSS 的点灯方案数，则有：

ans=∑S⊂Uf(S)×(−1)∣S∣ans=\sum_{S\subset U}f(S)\times (-1)^{|S|} ans=S⊂U∑​f(S)×(−1)∣S∣

这样计算出的 ansansans 是覆盖了除最左与最右两点之外的所有点但不覆盖最左最右点的方案数，只要加上 −∞-\infty−∞ 与 −∞-\infty−∞ 两个点即可。

我们考虑如何计算 f(S)f(S)f(S)。不覆盖到关键点集合 SSS 等价于用 SSS 中的位置将数轴划分为 ∣S∣+1|S|+1∣S∣+1 个区间，每个区间都不覆盖边界。我们可以 O(nm2)O(nm^2)O(nm2) 预处理出 g(l,r)g(l,r)g(l,r) 表示 [pl,pr][p_l,p_r][pl​,pr​] 区间内的电灯都不覆盖 plp_lpl​ 与 prp_rpr​ 关键点的方案数。 则有：

f(S)=∏i=1∣S∣−1g(Si,Si+1)f(S)=\prod\limits_{i=1}^{|S|-1}g(S_i,S_{i+1}) f(S)=i=1∏∣S∣−1​g(Si​,Si+1​)

所以容斥复杂度为 O(2mm)O(2^mm)O(2mm)。

ggg 数组的处理是平凡的。枚举所有灯 iii，并枚举所有包含它的区间 jjj。令灯位置为 aia_iai​，区间左右端点分别为 ljl_jlj​ 与 rjr_jrj​。则这盏灯的 power 范围是 [0,min⁡{ai−lj,rj−ai})[0,\min\{a_i-l_j,r_j-a_i\})[0,min{ai​−lj​,rj​−ai​})。给 jjj 的 ggg 乘上它的贡献即可。复杂度 O(nm2)O(nm^2)O(nm2)。

至此，我们 O(nm2)−O(2mm)O(nm^2)-O(2^mm)O(nm2)−O(2mm) 解决了静态问题。

考虑动态加灯。这盏灯对 ggg 的修改是可以 O(m2)O(m^2)O(m2) 解决的。但是如果每次更改后重新进行容斥，那么复杂度是不能接受的。于是一种方式是在动态加点前算出每个 ggg 值对答案的贡献，这样单次询问是 O(m2)O(m^2)O(m2) 的，可以通过，复杂度 O(nm2)−O(2mm)−O(qm2)O(nm^2)-O(2^mm)-O(qm^2)O(nm2)−O(2mm)−O(qm2)。

但是有另一种更为简介的方法可以在更优时间复杂度内（单次询问相同，预处理更优）解决此问题。

我们观察到每个容斥状态的值是所有子段值的连乘且所有子段不交且覆盖了区间 (−∞,+∞)(-\infty,+\infty)(−∞,+∞)，所以可以将容斥写成 dp 状物，即：

dpi=∑j=1i−1(−1)×dpj×g(j,i)dp_i=\sum_{j=1}^{i-1}(-1)\times dp_j\times g(j,i) dpi​=j=1∑i−1​(−1)×dpj​×g(j,i)

那么容斥系数在每加一段后都会 ×−1\times -1×−1，最后就是 (−1)∣S∣(-1)^{|S|}(−1)∣S∣。而 ∏g\prod g∏g 也在 dp 转移中进行了计算。最后答案为 dpm+1dp_{m+1}dpm+1​。这样单次 dp 复杂度是 O(m2)O(m^2)O(m2)，可以通过此题。

ABC280G Do Use Hexagon Grid 2

在一个六边形网格上，有 nnn 个点，同时给定一个 ddd，求有多少个点集，使得集合内的每对点的距离都不超过 ddd。n≤300n\le 300n≤300。

首先考虑从 (0,0)(0,0)(0,0) 到点 (x,y)(x,y)(x,y) 的距离怎么算。由于 (x,y)(x,y)(x,y) 可以向四方向走，可以向 (1,1)(1,1)(1,1) 和 (−1,−1)(-1,-1)(−1,−1) 方向走，所以距离是 max⁡(∣x∣,∣y∣,∣x−y∣)\max(|x|,|y|,|x-y|)max(∣x∣,∣y∣,∣x−y∣)。考虑将一个点 (x,y)(x,y)(x,y) 直接映射成一个三元组 (x,y,x−y)(x,y,x-y)(x,y,x−y)。那么问题就转化为：在一个三维空间内，有多少个点集是在同一个 (d+1)×(d+1)×(d+1)(d+1)\times (d+1)\times (d+1)(d+1)×(d+1)×(d+1) 的立方体内。

考虑钦定前两维的最小值位置，然后双指针去扫第三维在范围内的数。即钦定所有点的前两维为 [x,x+d],[y,y+d][x,x+d],[y,y+d][x,x+d],[y,y+d]。

但是这样会算重，所以考虑做一个小容斥，即 (x,y)−(x+1,y)−(x,y+1)+(x+1,y+1)(x,y)-(x+1,y)-(x,y+1)+(x+1,y+1)(x,y)−(x+1,y)−(x,y+1)+(x+1,y+1)。

最终复杂度 O(n3log⁡n)O(n^3\log n)O(n3logn)，log⁡n\log nlogn 由于要排序。
