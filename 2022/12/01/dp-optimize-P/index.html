CF809D Hitchhiking in the Baltic States

给你每个数的取值范围，让你求可能的 LIS 长度。

神仙题，做过的第一道平衡树优化 dp。一开始 dp 状态就设计错了。

我们令 fif_ifi​ 表示长度为 iii 的 LIS 的最小的结尾值。那么假如一个元素时，分成两类转移：

fi=min⁡{fi,l} (fi−1<l)fi=min⁡{fi,fi−1+1} (l≤fi−1<r)f_i=\min\{f_i,l\}\ (f_{i-1}<l)\\ f_i=\min\{f_i,f_{i-1}+1\}\ (l\le f_{i-1}<r) fi​=min{fi​,l} (fi−1​<l)fi​=min{fi​,fi−1​+1} (l≤fi−1​<r)

然后发现这个 fff 是严格单调递增的，用平衡树维护一下就好了，这个 fhq treap 就非常好写。

P3780 [SDOI2017]苹果树

首先原题那个 t−h≤kt-h\le kt−h≤k 的条件可以转化为最多取 ttt 个点，然后在免费取一条链（显然你会把这条链取到底部）。

然后考虑这个树形依赖不是很好处理，可能父亲节点不去完当前节点就取了。

于是把每个值不为 111 的节点拆成两个节点，个数分别为 111 和 au−1a_u-1au​−1。把 au−1a_u-1au​−1 的节点挂在 111 的节点下，这样就满足树形依赖了。

然后我们记下树的后序遍历，在后序遍历上做多重背包，再把每个节点的邻接表翻转，再在后序遍历上做多重背包。

我们发现一棵树除了一条链外，其他两个部分可以被看作第一个后序遍历上的一个前缀和第二个后序遍历上的一个前缀。

于是枚举一下免费取的那条链，把连边的背包信息合并起来取个 max⁡\maxmax 即可。

多重背包需要单调队列优化，复杂度 O(nk)O(nk)O(nk)。

P1399 [NOI2013] 快餐店

朴素做法是枚举割掉环上的哪条边，然后把剩下的树的直径求出来，然后所有的直径取个 min⁡\minmin，再除二即可。

因此我们就要快速算出割掉环上的每条边后树的直径。

直径有两种可能，第一种是不跨环的，第二种是跨环的。不跨换的可以直接算出。

而跨换的可以预处理出每个子树的深度，然后直接破环成链。枚举完边后，直径可能都在左边，都在右边，或左右各一个。

因此只要预处理出每个前缀取两个子树的距离 max⁡\maxmax，每个后缀取两个子树的距离 max⁡\maxmax，与每个前后缀取一个点的 max⁡\maxmax。

复杂度线性就做完了。

CF372C Watching Fireworks is Fun

单调队列优化 dp 板板题，每相邻两次烟火表演之间正着倒着做两遍 dp 就好了。

fi,jf_{i,j}fi,j​ 表示第 iii 次表演后，在位置 jjj 损失的最少喜悦值。

大于小于号写错，调了好久。

CF1428G2 Lucky Numbers (Hard Version)

神仙题，琢磨了 1h 周康阳的 sol，终于把他解释清楚了。

首先有结论每个数 nnn 都可以被拆成一堆只有 0,3,6,90,3,6,90,3,6,9 构成的数与至多一个含有其他数字的数。

证明考虑按位，如果这一位有一堆 0,3,6,90,3,6,90,3,6,9 与两个其他数字 a,ba,ba,b，如果 a+b≤9a+b\le9a+b≤9 那么就能变成 000 与 a+ba+ba+b，否则变成 999 与 a+b−9a+b-9a+b−9。

于是我们先考虑每个数只被分成一个数时，他的贡献，这个计算是平凡的。

然后我们考虑加一堆由 0,3,6,90,3,6,90,3,6,9 构成的数。考虑背包。

对于一个十进制位 iii，我们可以把他拆成 3(k−1)3(k-1)3(k−1) 个体积为 3×10i3\times 10^i3×10i，价值为 FiF_iFi​ 的物品。

这样进行背包一定是合法的，于是本题就做完了。

但是数据范围较大，不能直接背包。这是一个多重背包，可以考虑二进制分组优化。

P4093 [HEOI2016/TJOI2016]序列

很容易想到本题就是求一个最长三位偏序子序列长度，那么直接树套树优化一下 dp 即可。

但是这样很难写，考虑 cdq 分治，这样可以在相同时间复杂度内完成该问题，O(nlog⁡2n)O(n\log^2n)O(nlog2n)。

同样的 cdq 分治优化 dp 的题目还有P2487 [SDOI2011]拦截导弹。

CF1067D Computer Game

不难发现我们升级一定会选 bipib_ip_ibi​pi​ 最高的，升级成功后就一直选这个游戏获取收益。

那么我们不难想到 dp，令 ftf_tft​ 表示还剩 ttt 秒，还一个都没有升级，能获得的最大期望收益是多少。

我们令 M=max⁡{bipi}M=\max\{b_ip_i\}M=max{bi​pi​}，那么就能写出转移方程：

fi=max⁡{pj(aj+(i−1)M)+(1−pj)fi−1}f_i=\max\{p_j(a_j+(i-1)M)+(1-p_j)f_{i-1}\} fi​=max{pj​(aj​+(i−1)M)+(1−pj​)fi−1​}

分别表示是否升级成功。我们吧转移方程化简一下，得到：

fi=max⁡{pj((i−1)M−fi−1)+pjaj}+fi−1f_i=\max\{p_j((i-1)M-f_{i-1})+p_ja_j\}+f_{i-1} fi​=max{pj​((i−1)M−fi−1​)+pj​aj​}+fi−1​

然后令 si=(iM−fi)s_i=(iM-f_i)si​=(iM−fi​)，那么转移方程可以简化为：

fi=max⁡{si−1pj+ajpj}+fi−1f_i=\max\{s_{i-1}p_j+a_jp_j\}+f_{i-1} fi​=max{si−1​pj​+aj​pj​}+fi−1​

然后我们发现 sis_isi​ 是单调不减的，证明可以做一个差分。

所以我们将所有游戏按照 pip_ipi​ 为第一关键字，ajpja_jp_jaj​pj​ 为第二关键字排序，排完序后发现取得 jjj 一定是递增的。

然后这个转移是可以矩阵乘法优化的，每次一个转移完后找到下一个边界。

直接二分找边界复杂度是 O(nlog⁡2t)O(n\log^2t)O(nlog2t) 的，不能接受，因此可以考虑用倍增，于是就解决了。

CF739E Gosha is hunting

首先考虑每个神奇宝贝不可能同时有两个及以上的宝贝球去抽，也不可能同时有两个及以上超级球去抽。

因为抽一个多次还不如抽另一个，这样增加的期望更多。

于是我们令 fi,j,kf_{i,j,k}fi,j,k​ 表示处理了前 iii 个超级宝贝，用了 jjj 个宝贝球，kkk 个超级球时的最大期望个数。

那么转移很容易：

fi,j,k=max⁡{fi−1,j,k,fi−1,j−1,k+pi,fi−1,j,k−1+ui,fi−1,j−1,k−1+pi+ui−piui}f_{i,j,k}=\max\{f_{i-1,j,k},f_{i-1,j-1,k}+p_i,f_{i-1,j,k-1}+u_i,f_{i-1,j-1,k-1}+p_i+u_i-p_iu_i\} fi,j,k​=max{fi−1,j,k​,fi−1,j−1,k​+pi​,fi−1,j,k−1​+ui​,fi−1,j−1,k−1​+pi​+ui​−pi​ui​}

分别表示啥都不选，选一个宝贝球，选一个超级球，选一个宝贝球和一个超级球。

然后发现对于固定的 i,ji,ji,j，fi,j,kf_{i,j,k}fi,j,k​ 是关于 kkk 的凸函数，因此考虑 wqs 二分。

通过这道题终于明白了 wqs 二分二分的其实是去切 dp 值凸函数的那个一次函数的斜率。

理解之后就是 wqs 二分板板题了。

P3522 [POI2011]TEM-Temperature

给你每个数的取值范围，让你求可能的最长非递减子段长度。

子序列变成了子段，简单了不少。

不难发现一个区间合法，当且仅当对于任意位置 iii，都有 Li≤min⁡Rj(j≥i)L_i\le\min R_j(j\ge i)Li​≤minRj​(j≥i)。

然后我们想求出对于每个 rrr 的最左边能到达的 lll，发现这个东西是由单调性的。

于是套上决策单调性模板即可，再写一个 ST 表。

但是这题可以用单调栈做到线性，想法也非常简单。

HDU 7221. Darkmoon Faire

线段树优化 dp。（或者说线段树辅助 dp 更加贴切？）

我们用 h(l,r)h(l,r)h(l,r) 来表示序列 {al,al+1,…,ar}\{a_l,a_{l+1},\dots,a_{r}\}{al​,al+1​,…,ar​} 是否是一个傻子序列。

令 fif_ifi​ 表示 [l,n][l,n][l,n] 这部分划分成多个傻子序列有多少种方法。那么暴力 dp 很好想，即：

fi=∑j=i+1nfj×h(i,j−1)f_i=\sum_{j=i+1}^n f_j\times h(i,j-1) fi​=j=i+1∑n​fj​×h(i,j−1)

接下来哦我们呢就要考虑怎么在 dp 转移的同时快速算出 符合 h(i,j−1)=1h(i,j-1)=1h(i,j−1)=1 的位置的 fff 值之和。

我们想到要满足 h(l,r)=1h(l,r)=1h(l,r)=1 有两个条件，即：最大值在奇数位 与 最小值在偶数位。当我们枚举 iii 时，起点就确定了，是 i+1i+1i+1，那么总体奇数位的最大值会对奇数位的起点产生一个合法条件的贡献，偶数位的最大值会对偶数位的起点产生一个合法条件的贡献。同样，奇数位的最小值会对偶数位的起点产生一个合法条件的贡献，偶数位的最小值会对奇数位的起点产生一个合法条件的贡献。而所有两个合法条件的位置 iii 会对总答案产生 fif_ifi​ 的贡献。

因此，我们用两个单调栈维护可能的最大值与最小值，并用两棵线段树维护每个位置的合法条件个数即可。复杂度 O(nlog⁡n)O(n\log n)O(nlogn)。

0810 NOIP 模拟 T3

听说有贪心解法，但是不会证明正确性，于是还是老老实实写 Ex Slope Trick。

我们令 fi,jf_{i,j}fi,j​ 表示前 iii 个数，其中 bi=jb_i=jbi​=j 时的最小代价。

很容易得到朴素的 dp 方程：

fi,j=min⁡k=1jfi−1,k+(ai−j)2f_{i,j}=\min_{k=1}^j f_{i-1,k}+(a_i-j)^2 fi,j​=k=1minj​fi−1,k​+(ai​−j)2

我们令：

gi,j=min⁡k=1jfi−1,kg_{i,j}=\min\limits_{k=1}^j f_{i-1,k} gi,j​=k=1minj​fi−1,k​

于是有：

fi,j=gi−1,j+(ai−j)2f_{i,j}=g_{i-1,j}+(a_i-j)^2 fi,j​=gi−1,j​+(ai​−j)2

看起来很像 Slope Trick 的模板，不失一般性的，我们令：

Fi(x)=fi,xGi(x)=gi,xHi(x)=(ai−x)2F_i(x)=f_{i,x}\\ G_i(x)=g_{i,x}\\ H_i(x)=(a_i-x)^2 Fi​(x)=fi,x​Gi​(x)=gi,x​Hi​(x)=(ai​−x)2

那么就有：

Fi(x)=Gi−1(x)+Hi(x)F_i(x)=G_{i-1}(x)+H_i(x) Fi​(x)=Gi−1​(x)+Hi​(x)

已经很有 Slope Trick 的味了。

但是 Slope Trick 中所定义的“美妙的函数”本质是一个斜率单调递增的分段一次函数，但是本题中三个函数都是分段二次函数。

因此，我们对三个函数分别求导，他们的导函数就是斜率单调递减的“美妙的函数”啦！

先来看 Hi′(x)=2x−2aiH_i'(x)=2x-2a_iHi′​(x)=2x−2ai​，我们发现有一个 222 的系数，于是全部提取。

那么每次 F′→G′F'\rightarrow G'F′→G′ 的变换就是将一个斜率单调递减的“美妙的函数”与一个斜率为 111 的一次函数相加，并将 G′(x)>0G'(x)>0G′(x)>0 的部分全部压到 000 上。此处请读者自行手模理解。

手模后，我们发现操作中的变换分为两种情况：

Case 1

上图中，H′H'H′ 与 G′G'G′ 有交点（且一定至少有一个交点在 xxx 轴上），那么变换之后相当于加了一段 Slope 为 111 的一次函数，并将前面所有段的 Slope +1+1+1。

Case 2

上图中，H′H'H′ 与 G′G'G′ 没有交点，那么我们要找到他们相加后第一个 =0=0=0 的点的位置。我们可以将 H′×−1H'\times -1H′×−1，那么这个函数与 G′G'G′ 的交点横坐标就是第一个 =0=0=0 的点的横坐标，就能快速计算 F′F'F′ 了。

每次处理完后第一个导函数 =0=0=0 的位置的横坐标即为当前的最优选择。

为了维护这个函数 G′G'G′，我们将每个转折点的横坐标与它之前的那一段函数的 Slope 记录下来，遇到 Case 1 则直接加一段，遇到 Case 2 就将之前的段依次弹出直到寻找到与其有交的点即可。每次操作完后 Slope 均 +1+1+1（由于所有系数均已 ÷2\div 2÷2）。这样是 O(n2)O(n^2)O(n2) 的。

我们用一个 LazyTag 维护一下所有 Slope 所加的值就可以做到线性。

具体细节请读者自行推导。

P3746 [六省联考 2017] 组合数问题

首先考虑暴力 dp，令 fi,jf_{i,j}fi,j​ 表示 ∑x=0∞(ixk+j)\sum\limits_{x=0}^\infty \binom{i}{xk+j}x=0∑∞​(xk+ji​)，即 iii 个数中，选出  mod k=j\bmod k=jmodk=j 个数的方案数。

发现可以从 fif_ifi​ 和 fjf_jfj​ 转移到 fi+jf_{i+j}fi+j​，具体地，枚举前 iii 个数选  mod k\bmod kmodk 为几个，后 jjj 个数选  mod k\bmod kmodk 为几个，然后乘起来求和即可。

然后发现这个性质可以类似快速幂的倍增优化，于是在 O(k2log⁡n)O(k^2\log n)O(k2logn) 的时间复杂度内做完了。
